Plately - v 2.0.0

Plately is a simple starting template I use for front-end web development projects. It's set up to quickly let developers start creating with Jade, Sass, jQuery, and any other dependencies they need to add.

Requirements

This project has a few basic global dependencies to make it.

  • Node.js
  • Gulp.js, a node task runner for automating common tasks
  • NPM (node package manager)
  • Bower
  • Basic understanding of Jade, Sass, jQuery, etc.

Folder Structure

Folders that are already there:

  • Jade: Where all the Jade files are stored, which are converted into HTML for the project.
  • Img: This is in the Jade folder, and is where all your images go.
  • JS: Where your jQuery and Javacript files go, which is the interactivity and logic for the project. Structured to better organize the different files.
  • Public: Static pages that contain the Sass and JS documentation that makes it easy to read through and understand what makes up the project.
  • Sass: Where all the Sass files go, separated by a folder structure for better organization.

Folders that are created:

These are created with the commands in the "How to Set Up" section below.

  • Build: The folder that ultimately contains all your final project files - HTML, CSS, Javascript, images, the whole thing.
  • Components: All the external dependencies for the project, such as libraries and frameworks. Can be installed and managed with Bower.
  • node_modules: All the npm dependencies that put the project together. Take a long time to install but are vital to creating anything at all with Plately.

NPM Dependencies

  • express: A web application framework that currently just puts the site on localhost:3000 while gulp is running.
  • gulp: Basic Gulp module for running the workflow.
  • gulp-autoprefixer: Automatically adds browser prefixes to certain CSS styles to ensure cross-compatibility.
  • gulp-concat: For merging different JS files together from the JS folder into a single one for the Assets folder.
  • gulp-imagemin: For optimizing images in the images placed in the Jade/Img folder and sent to Build.
  • gulp-include: for combining and minifying JS files.
  • gulp-jade: For converting Jade files to HTML.
  • gulp-livereload: For using the Chrome LiveReload extension so pages refresh automatically with changes.
  • gulp-notify: Notifying of completed processes or errors.
  • gulp-plumber: So any mistakes send a message without halting the workflow.
  • gulp-rename: Renaming any files before sending them to the Build folder.
  • gulp-sass: For converting Sass to CSS.
  • gulp-sourcemaps: Adding sourcemaps to CSS and JS files to speed up debugging.
  • gulp-uglify: For compressing JS files.
  • gulp-uncss: Removes any CSS element selectors that aren't being used in the Build folder.
  • jsdoc: For automated documentation of JS code, such as functions.
  • sassdoc: For automated documentation of Sass functions, mixins, variables, etc.

Bower Dependencies

  • Bourbon: A lightweight library of Sass mixins for extra functionality.
  • Neat: A grid system for responsive layouts using Bourbon and with solid range of layout functionality.
  • jQuery: A Javascript library that makes writing it and using it a couple hundred times easier.

How to Set Up

There are several commands to get all the needed dependencies installed and ready to view.

npm install
  • This install all the node dependencies in the newly created node_modules folder.
bower install
  • This creates the "components" folder and installs all the components controlled with Bower.
gulp
  • This is the basic command that constructs the Build folder. It turns the Jade into HTML, the Sass into CSS, optimizes the images, and puts together all the Javascript and jQuery files. This also runs in the background and updates the Build folder any time there's a saved change to any of these files.
  • While this is running, you can also view the project on a browser on localhost:3000.

Gulp Commands

These are other gulp commands to run from the terminal for building the project. Most of them are the individual commands run as part of the default "gulp" command, in case you don't need to run all at once.

gulp scripts
  • This only merges and minifies your JS files.
gulp sass
  • This only processes and converts your Sass files.
gulp uncss
  • This runs through all your Build's CSS, compares it to your HTML, and removes any CSS that's not being used.
gulp sassdoc
  • This creates any documentation from inline Sass comments and updates it in the Public folder.
gulp jsdoc
  • This does the same as Sassdoc, except for the JS files and their inline comments.
gulp images
  • This goes through all the images in the Img folder, optimizes them to reduce size, and sends them to the build folder.
gulp jade
  • This converts all your Jade files to HTML and sends them to the build folder.
gulp docwatch
  • This creates the documentation for your Sass and JS files, and allows for live updating of the documentation as you write and save more inline comments to the files.
gulp prod
  • This is for when your project is ready to be launched. It updates your Sass and JS documentation and optimizes your CSS through minifying it, removing sourcemaps, and adding vendor prefixes.
